# Software Testing Life Cycle (STLC) – Saucedemo Testing Project

This document describes the STLC phases followed in this QA project.

---

## 1. Requirement Analysis
### Activities:
- Studied core features of Saucedemo (Login, Cart, Checkout, etc.)
- Identified what can be tested and what is not available.
- Understood UI behaviour, validations, and user flows.

### Deliverables:
- Requirements Document  
- List of testable features  

---

## 2. Test Planning
### Activities:
- Decided scope of testing (manual functional testing).
- Defined resources: tester, Chrome browser, Saucedemo demo site.
- Selected test techniques (positive, negative, UI testing).
- Estimated number of test cases.

### Deliverables:
- Test Plan (in README)  
- Test Strategy decisions  
- Effort estimation  

---

## 3. Test Case Design
### Activities:
- Created detailed test cases for:
  - Login / Logout  
  - Product Listing  
  - Add to Cart  
  - Cart & Remove  
  - Checkout  
  - Price calculation  

### Deliverables:
- Test cases (TC-001 to TC-010)  
- Use cases (UC-01 to UC-05)  

---

## 4. Test Environment Setup
### Activities:
- Prepared Chrome browser for testing.
- Created folder structure with:
  - evidence/screenshots  
  - .md documentation  
- Verified website availability.

### Deliverables:
- Test Environment Checklist  
- Ready test workspace  

---

## 5. Test Execution
### Activities:
- Executed all test cases manually.
- Marked each as **Pass / Fail**.
- Collected screenshots for evidence.
- Logged UI issues found.

### Deliverables:
- Executed Test Case file  
- Evidence screenshots  

---

## 6. Defect Reporting & Tracking
### Activities:
- Reported bugs using bug report template:
  - BUG-001 – Login error message missing  
  - BUG-002 – Filter dropdown not clickable
  - BUG-003 - Missing payment  
  - BUG-004 – Price mismatch due to floating-point issue  

### Deliverables:
- Individual Bug Reports  
- Failed Test Case references  

---

## 7. Test Closure
### Activities:
- Prepared Summary Reports for:
  - Login  
  - Product Listing  
  - Add to cart
  - Payment
  - Checkout  
- Ensured all high-priority bugs are documented.
- Organized final documentation for submission.

### Deliverables:
- Test Summary Reports   
- Observations / Recommendations
